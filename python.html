Java Introduction

Java is a high-level, object-oriented programming language known for its portability and robustness.

HelloWorld.java 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Explanation: This is the simplest Java program. The main method is the entry point of any Java application. System.out.println prints text to the console.

Real-life Example

Java is widely used in enterprise applications, Android app development, and large-scale systems like banking applications.

Operators

Java provides various types of operators for performing operations on variables and values.

Operators.java 
public class Operators {
    public static void main(String[] args) {
        int a = 10, b = 5;
        
        // Arithmetic operators
        System.out.println("a + b = " + (a + b)); // 15

        System.out.println("c = " + c); // 15

Explanation: Java operators include arithmetic (+, -, *, /, %), relational (==, !=, >, <), logical (&&, ||, !), and assignment (=, +=, -=) operators.

Loops

Execute a block of code repeatedly with loops.

Loops.java 
public class Loops {
    public static void main(String[] args) {
        // for loop
        System.out.println("For loop:");
        for (int i = 1; i <= 5; i++) {
            System.out.println("Count: " + i);
        }
        
        // while loop
        System.out.println("\nWhile loop:");
        int j = 1;
        while (j <= 5) {
            System.out.println("Count: " + j);

        // do-while loop
        System.out.println("\nDo-while loop:");

        System.out.println("\nFor-each loop:");
        int[] numbers = {1, 2, 3, 4, 5};
        for (int num : numbers) {
            System.out.println("Number: " + num);
        }
    }
}

Explanation: Java provides several looping mechanisms: for loops for known iterations, while loops for condition-based iterations, do-while loops that execute at least once, and for-each loops for iterating over arrays/collections.

Methods

Organize your code into reusable blocks called methods.

Methods.java 
public class Methods {
    // Method with no parameters and no return value
    static void greet() {
        System.out.println("Hello, World!");
    }
    
    // Method with parameters and return value
    static int add(int a, int b) {
        return a + b;
    }
    
    // Method overloading (same name, different parameters)
    static int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Variable arguments (varargs)
    static int sum(int... numbers) {
        int total = 0;
        for (int num : numbers) {
            total += num;
        }
        return total;
    }
    
    public static void main(String[] args) {
        greet();
        System.out.println("Sum of 5 and 3: " + add(5, 3));
        System.out.println("Sum of 1, 2 and 3: " + add(1, 2, 3));
        System.out.println("Sum of 1, 2, 3, 4, 5: " + sum(1, 2, 3, 4, 5));
    }
}

Explanation: Methods are blocks of code that perform specific tasks. They can accept parameters, return values, and be overloaded (same name, different parameters). Varargs allow methods to accept variable numbers of arguments.

OOP Concepts

Object-Oriented Programming concepts in Java: Classes, Objects, Inheritance, Polymorphism, Abstraction, and Encapsulation.

OOPConcepts.java 
// Abstraction using abstract class
abstract class Animal {
    // Encapsulation using private fields with public getters/setters
    private String name;
    
    public Animal(String name) {
        this.name = name;
    }
    
    public String getName() {
        return name;
    }
    
    // Abstract method (no implementation)
    public abstract void makeSound();
    
    // Concrete method
    public void sleep() {
        System.out.println(name + " is sleeping");
    }
}

// Inheritance
class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }
    
    // Polymorphism: method overriding
    @Override
    public void makeSound() {
        System.out.println(getName() + " says: Woof!");
    }
}

class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(getName() + " says: Meow!");
    }
}

public class OOPConcepts {
    public static void main(String[] args) {
        // Polymorphism: using parent reference for child objects
        Animal myDog = new Dog("Buddy");
        Animal myCat = new Cat("Whiskers");
        
        myDog.makeSound();
        myCat.makeSound();
        
        myDog.sleep();
        myCat.sleep();
    }
}

Explanation: Java is an object-oriented language. Key OOP concepts include:

  • Encapsulation: Bundling data and methods that operate on that data
  • Inheritance: Creating new classes from existing ones
  • Polymorphism: Ability to take many forms (method overriding)
  • Abstraction: Hiding implementation details

Exception Handling

Handle runtime errors gracefully using exception handling.

ExceptionHandling.java 
import java.util.InputMismatchException;
import java.util.Scanner;

public class ExceptionHandling {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        
        try {
            System.out.print("Enter a number: ");
            int num = scanner.nextInt();
            
            System.out.print("Enter another number: ");
            int divisor = scanner.nextInt();
            
            int result = num / divisor;
            System.out.println("Result: " + result);
            
        } catch (InputMismatchException e) {
            System.out.println("Error: Please enter valid integers");
        } catch (ArithmeticException e) {
            System.out.println("Error: Cannot divide by zero");
        } catch (Exception e) {
            System.out.println("An unexpected error occurred: " + e.getMessage());
        } finally {
            System.out.println("This block always executes");
            scanner.close();
        }
        
        // Custom exception
        try {
            validateAge(15);
        } catch (InvalidAgeException e) {
            System.out.println("Caught custom exception: " + e.getMessage());
        }
    }
    
    static void validateAge(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Age must be 18 or older");
        }
        System.out.println("Valid age: " + age);
    }
}

// Custom exception class
class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

Explanation: Exception handling allows you to manage runtime errors gracefully. Use try blocks for code that might throw exceptions, catch blocks to handle specific exceptions, and finally blocks for code that always executes. You can also create custom exceptions.

File Handling

Read from and write to files in Java.

FileHandling.java 
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.FileReader;
import java.io.FileWriter;
import java.io.IOException;

public class FileHandling {
    public static void main(String[] args) {
        // Writing to a file
        try (BufferedWriter writer = new BufferedWriter(new FileWriter("example.txt"))) {
            writer.write("Hello, Java File Handling!");
            writer.newLine();
            writer.write("This is a second line.");
            System.out.println("Successfully wrote to the file.");
        } catch (IOException e) {
            System.out.println("An error occurred while writing to the file: " + e.getMessage());
        }
        
        // Reading from a file
        try (BufferedReader reader = new BufferedReader(new FileReader("example.txt"))) {
            String line;
            System.out.println("\nReading from file:");
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            System.out.println("An error occurred while reading the file: " + e.getMessage());
        }
    }
}

Explanation: Java provides several classes for file I/O operations. FileWriter and BufferedWriter are used for writing to files, while FileReader and BufferedReader are used for reading from files. The try-with-resources statement automatically closes the resources.

Advanced Topics

Multithreading, generics, and other advanced Java concepts.

AdvancedJava.java 
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

// Multithreading
class MyRunnable implements Runnable {
    private String name;
    
    public MyRunnable(String name) {
        this.name = name;
    }
    
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println(name + ": " + i);
            try {
                Thread.sleep(500);
            } catch (InterruptedException e) {
                System.out.println(name + " interrupted");
            }
        }
        System.out.println(name + " finished");
    }
}

// Generics
class GenericBox {
    private T content;
    
    public void setContent(T content) {
        this.content = content;
    }
    
    public T getContent() {
        return content;
    }
}

public class AdvancedJava {
    public static void main(String[] args) {
        // Multithreading with ExecutorService
        System.out.println("Multithreading example:");
        ExecutorService executor = Executors.newFixedThreadPool(2);
        
        executor.execute(new MyRunnable("Thread 1"));
        executor.execute(new MyRunnable("Thread 2"));
        
        executor.shutdown();
        
        // Generics example
        System.out.println("\nGenerics example:");
        GenericBox stringBox = new GenericBox<>();
        stringBox.setContent("Hello Generics");
        System.out.println("String box: " + stringBox.getContent());
        
        GenericBox intBox = new GenericBox<>();
        intBox.setContent(123);
        System.out.println("Integer box: " + intBox.getContent());
    }
}

Explanation: Advanced Java concepts include:

  • Multithreading: Executing multiple threads simultaneously for better performance
  • Generics: Enabling types (classes and interfaces) to be parameters when defining classes, interfaces, and methods
  • Executors: Higher-level API for working with threads

OOP Concepts

Object-Oriented Programming concepts in Python: Classes, Objects, Inheritance, Polymorphism, Abstraction, and Encapsulation.

oop.py 
# Basic class
class Dog:
    # Class attribute
    species = "Canis familiaris"
    
    # Initializer (constructor)
    def __init__(self, name, age):
        # Instance attributes
        self.name = name
        self.age = age
    
    # Instance method
    def description(self):
        return f"{self.name} is {self.age} years old"
    
    # Another instance method
    def speak(self, sound):
        return f"{self.name} says {sound}"

# Creating objects
my_dog = Dog("Buddy", 5)
print(my_dog.description())
print(my_dog.speak("Woof!"))

# Inheritance
class Bulldog(Dog):  # Inherits from Dog
    def run(self, speed):
        return f"{self.name} runs at {speed} mph"
    
    # Method overriding
    def speak(self, sound="Woof Woof!"):
        return super().speak(sound)

my_bulldog = Bulldog("Max", 3)
print(my_bulldog.run(10))
print(my_bulldog.speak())

# Encapsulation with private attributes
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.__balance = balance  # Private attribute
    
    def deposit(self, amount):
        self.__balance += amount
        print(f"Deposited ${amount}. New balance: ${self.__balance}")
    
    def withdraw(self, amount):
        if amount > self.__balance:
            print("Insufficient funds")
        else:
            self.__balance -= amount
            print(f"Withdrew ${amount}. New balance: ${self.__balance}")
    
    # Getter method
    def get_balance(self):
        return self.__balance

account = BankAccount("John", 1000)
account.deposit(500)
account.withdraw(200)
# print(account.__balance)  # This would cause an error
print(f"Balance: ${account.get_balance()}")

# Abstract base class
from abc import ABC, abstractmethod

class Animal(ABC):
    @abstractmethod
    def speak(self):
        pass

class Cat(Animal):
    def speak(self):
        return "Meow!"

my_cat = Cat()
print(my_cat.speak())

Explanation: Python supports object-oriented programming with:

  • Classes and Objects: Blueprints for creating objects
  • Inheritance: Creating new classes from existing ones
  • Encapsulation: Bundling data and methods (using private attributes with __)
  • Polymorphism: Ability to take many forms (method overriding)
  • Abstraction: Hiding implementation details (using abstract base classes)

exception_handling.py 
# Basic exception handling
try:
    num = int(input("Enter a number: "))
    result = 10 / num
    print(f"Result: {result}")
except ValueError:
    print("Error: Please enter a valid integer")
except ZeroDivisionError:
    print("Error: Cannot divide by zero")
except Exception as e:
    print(f"An unexpected error occurred: {e}")
else:
    print("Division completed successfully")
finally:
    print("This block always executes")

# Raising exceptions
def validate_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative")
    if age < 18:
        raise ValueError("Age must be 18 or older")
    return f"Valid age: {age}"

try:
    print(validate_age(25))
    print(validate_age(15))  # This will raise an exception
except ValueError as e:
    print(f"Caught exception: {e}")

# Custom exceptions
class InvalidEmailError(Exception):
    def __init__(self, email, message="Invalid email format"):
        self.email = email
        self.message = message
        super().__init__(self.message)
    
    def __str__(self):
        return f"{self.message}: {self.email}"

def validate_email(email):
    if "@" not in email or "." not in email:
        raise InvalidEmailError(email)
    return f"Valid email: {email}"

try:
    print(validate_email("john.example.com"))  # This will raise custom exception
except InvalidEmailError as e:
    print(e)

# Using assert for debugging
def calculate_average(numbers):
    assert len(numbers) > 0, "List cannot be empty"
    return sum(numbers) / len(numbers)

print(calculate_average([1, 2, 3, 4, 5]))
# print(calculate_average([]))  # This will raise AssertionError

Explanation: Python exception handling uses try, except, else, and finally blocks. You can raise exceptions with raise and create custom exception classes. The assert statement is used for debugging purposes to check if conditions are met.

# Decorators
def my_decorator(func):
    def wrapper():
        print("Something is happening before the function is called.")
        func()
        print("Something is happening after the function is called.")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()

# Generators
def countdown(n):
    while n > 0:
        yield n
        n -= 1

print("\nCountdown:")
for i in countdown(5):
    print(i)

# Multithreading
import threading
import time

def print_numbers():
    for i in range(1, 6):
        time.sleep(1)
        print(f"Number: {i}")

def print_letters():
    for letter in ['a', 'b', 'c', 'd', 'e']:
        time.sleep(1.5)
        print(f"Letter: {letter}")

print("\nMultithreading example:")
thread1 = threading.Thread(target=print_numbers)
thread2 = threading.Thread(target=print_letters)

thread1.start()
thread2.start()

thread1.join()
thread2.join()

# Context managers
class FileManager:
    def __init__(self, filename, mode):
        self.filename = filename
        self.mode = mode
        self.file = None
    
    def __enter__(self):
        self.file = open(self.filename, self.mode)
        return self.file
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.file:
            self.file.close()

print("\nContext manager example:")
with FileManager("test.txt", "w") as f:
    f.write("Context manager test")
    print("File written using context manager")
